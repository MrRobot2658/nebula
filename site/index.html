为什么是 Nebula

和传统营销 SaaS，根本不是一回事

微盟、有赞、尘锋、销售易仍是「菜单 - 表单 - 表格」范式。Nebula 借鉴 Marketo 的能力模型，但为中国市场做了三个关键改变。

💬

ChatUI 统一交互

用一个聊天界面替代传统表单后台。日常营销在对话中完成，复杂配置侧滑展开，零学习成本上手。

🤖

原生 AI Agent

AI 不是某个模块的附加功能，而是产品内核。实时分析客户意图、识别身份、生成话术与下一步建议。

🧩

Skill 插件化架构

渠道、内容、活动全部以 Skill 接入，可插拔可组合。小微企业最小只装 3 个，大企业全域编排。

统一数据层

所有 Skill 共享的数据底座

CDP + OneID + 事件总线，让全渠道客户与行为统一可用。

🗂️ CDP 客户数据平台

集成全渠道客户数据。

  • · 行为数据：浏览 / 点击 / 购买 / 咨询
  • · 属性数据：地域 / 行业 / 职级 / 企业
  • · 交易数据：金额 / 频次 / 品类偏好

🆔 OneID 身份统一

跨渠道身份自动关联。

  • · 微信 OpenID → 企微 → UnionID → 手机号
  • · 确定性匹配 + 概率匹配
  • · 冲突解决：优先级 + 人工合并

🔁 事件总线

驱动所有 Skill 协同。

  • · 发布/订阅，至少一次投递
  • · 事件持久化，支持回放
  • · 跨 Skill 延迟 < 100ms

完整案例

新客「李四」从微信咨询到下单

某 DTC 皮草品牌用 Nebula 管理营销，一次真实对话如何被自动放大为转化。

01 · message_received
微信消息到达

wechat-skill 发出事件，ChatUI 中栏显示对话，右栏自动识别客户身份。

02 · AI 分析 + 评分
意图识别，初始 60 分

AI 判定「产品咨询 · 新客」，建议：高意向新客，回复产品介绍 + 推送夏日优惠券。

03 · 快捷回复
素材一键发送

素材面板推荐产品 PDF / 新品图 / 优惠券，点击填入即发，报表记录渠道与活动归因。

04 · 自动化跟进
评分 93 > 阈值 80，触发流程

启动「新客跟进流程」：24h 后发优惠券提醒，7 天未下单自动回访。

05 · order_placed
下单转化 🎉

小程序下单成功，报表归因渠道=微信，评分 +20，OneID 关联微信身份到订单。
