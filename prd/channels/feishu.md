# 飞书 Feishu

> **Skill ID** `feishu-skill` ｜ **分类** 渠道 ｜ **文档版本** v0.2

## 一、模块定位

飞书是字节跳动旗下的一站式企业协作平台，以即时消息、文档、日历、审批等深度集成的办公能力著称，在互联网公司与新经济企业中渗透率较高。对于面向企业客户（B2B）开展营销与服务的团队而言，飞书既是内部协同的工作台，也是连接企业客户、推进协作式营销的沟通渠道。飞书渠道 Skill（`feishu-skill`）通过飞书开放平台接入，把交互式消息卡片、群机器人、审批、文档、日历与联系人等能力封装进 Nebula，使营销动作能够嵌入企业协作流并与团队工作场景无缝衔接。

本 Skill 解决 B2B 营销与服务在协作平台上"沟通、审批、内容、客户信息相互割裂"的问题。Nebula 把飞书能力作为标准技能接入统一数据层：群机器人收到的消息、审批的处理结果、文档的生成等都通过事件总线回流，并经 OneID 与客户主档关联；交互式消息卡片由原生 AI Agent 驱动生成与响应，让营销活动的审批、方案分发、日程协调都在飞书内高效完成，与钉钉 Skill 共同构成企业级协作渠道的覆盖。

## 二、目标用户与典型场景

- **目标用户**：面向企业客户的 B2B 市场与销售团队、需要走审批流的营销活动负责人、用飞书做内部协同与客户对接的运营团队。
- **典型场景**：
  - 营销活动方案通过飞书审批流逐级流转，审批结果实时回流并触发后续执行动作。
  - 群机器人按设定定时推送营销简报或在被关键词/@提及时自动回复，承担轻量化的协作通知。
  - 系统根据营销需求自动生成方案文档并在飞书内分发，同时用日历协调活动日程与提醒。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 交互式消息卡片 | 支持图文、按钮、表单等富交互卡片，让消息可承载操作与反馈。 |
| 群机器人 | 支持自动回复、定时推送与关键词触发，承担群内的自动化通知与互动。 |
| 审批 | 支持营销活动的审批流程，把审批环节嵌入营销执行链路。 |
| 文档 | 支持自动生成营销方案文档，沉淀并分发营销内容产出。 |
| 日历 | 支持营销活动的日程管理与提醒，协调团队与客户的时间安排。 |
| 联系人 | 同步组织架构并与客户进行关联，打通人与客户信息的映射。 |

## 四、ChatUI 交互

客户或团队通过飞书发来的消息、以及群机器人接收到的 @提及，实时进入 ChatUI 中栏会话流，运营在统一界面即可处理飞书侧的沟通；右栏客户卡片展示对方的飞书身份与组织架构信息，并经 OneID 与客户主档及其他渠道画像关联，呈现 B2B 场景下"人-企业-客户"的立体视图。中栏 AI 建议区结合会话上下文给出回复话术、可发送的交互卡片或可触发的审批/文档动作建议，运营一键采纳。右栏活动区展示该客户相关的审批处理、文档生成、机器人互动等事件时间线，使协作式营销过程清晰可追溯。

## 五、触发事件

| 事件 | 触发时机 |
| --- | --- |
| `message_received` | 收到客户或团队通过飞书发来的消息时触发。 |
| `approval_processed` | 营销相关审批被处理（通过/驳回）时触发。 |
| `bot_mentioned` | 群机器人被 @提及或被关键词触发时触发。 |
| `doc_created` | 营销方案等文档被自动生成或创建时触发。 |

## 六、配置参数

```json
{"api_key":"your_feishu_api_key","webhook_url":"https://your-domain.com/webhook","enabled":true}
```

## 七、关键指标

- 消息触达率与卡片交互率：衡量飞书消息与交互卡片的有效触达。
- 审批流转时长与通过率：衡量营销活动审批的效率。
- 机器人响应率：衡量群内自动化互动的覆盖与及时性。
- 文档生成与分发量：衡量营销内容产出与协作效率。
- 客户对接活跃度：衡量 B2B 协作渠道的客户连接质量。

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID / 事件总线）；飞书开放平台 API 用于消息卡片、机器人、审批、文档、日历、联系人等能力对接。
- **边界**：本 Skill 负责飞书渠道的连接、消息与协作事件采集，不负责营销内容的策划创作（方案正文质量交由内容模块），也不负责跨渠道自动化旅程的编排与活动策略设计（交由活动/自动化模块）。
