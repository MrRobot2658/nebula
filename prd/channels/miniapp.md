# 小程序 Mini Program

> **Skill ID** `miniapp-skill` ｜ **分类** 渠道 ｜ **文档版本** v0.2

## 一、模块定位

小程序是中国移动互联网最重要的轻量化商业载体，无需下载安装、即用即走的特性使其成为连接流量与交易的关键枢纽。无论是微信还是支付宝生态，小程序都承载了从商品浏览、下单支付到分享裂变的完整交易闭环，是 DTC 品牌实现私域成交与复购的转化主战场。小程序渠道 Skill（`miniapp-skill`）对接微信/支付宝小程序，把小程序商城、模板消息订阅、客服消息、用户授权、支付、分享裂变与数据分析等能力统一封装进 Nebula。

本 Skill 对标有赞微商城、微盟智慧零售等小程序电商方案，解决品牌在小程序运营中"交易数据与客户运营脱节"的痛点。Nebula 将小程序的下单、支付、分享、客服会话等行为实时接入事件总线，经 OneID 与公众号、企业微信等渠道的客户画像融合，使每一笔订单与每一次互动都成为可分析、可触发后续运营的客户事件；客服消息进入统一 ChatUI 并由原生 AI Agent 辅助应答，让小程序从单纯的交易容器升级为可运营、可转化、可复购的智能商业触点。

## 二、目标用户与典型场景

- **目标用户**：以小程序商城为转化主阵地的 DTC/零售品牌电商运营、做拼团砍价分销的增长运营、依托小程序提供会员与售后服务的客服团队。
- **典型场景**：
  - 用户在小程序商城下单支付，订单与支付事件实时回流，触发物流提醒与复购关怀流程。
  - 通过用户授权获取手机号与昵称头像，完成会员注册并与全域画像合并，沉淀可触达的客户资产。
  - 发起拼团/砍价/分销等分享裂变活动，追踪分享与拉新链路，评估社交裂变带来的增量转化。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 小程序商城 | 提供商品展示、购物车、下单与支付的完整交易能力，构成小程序成交闭环。 |
| 模板消息订阅 | 支持下单提醒、物流更新、活动通知等订阅消息的下发，完成交易后的关键触达。 |
| 客服消息 | 支持小程序内的客服会话，承接用户在交易过程中的咨询与售后。 |
| 用户授权 | 获取用户手机号、头像昵称等授权信息，用于会员注册与画像沉淀。 |
| 支付 | 集成微信支付与支付宝，支撑小程序内的在线交易与收款。 |
| 分享裂变 | 支持拼团、砍价、分销等社交玩法，驱动用户分享与低成本拉新。 |
| 数据分析 | 采集访问、转化、复购等数据，支撑小程序运营的量化决策。 |

## 四、ChatUI 交互

用户在小程序内发起的客服消息实时进入 ChatUI 中栏会话流，运营在统一界面接待咨询与售后，无需进入小程序后台来回切换；右栏客户卡片展示该用户的小程序身份与授权信息（手机号、昵称），并经 OneID 融合呈现其历史订单、复购情况与跨渠道画像，让客服在响应前即掌握客户的消费背景。中栏 AI 建议区结合用户的订单状态与商品上下文，自动给出回复话术、关联商品或可下发的订阅消息建议，运营一键采纳。右栏活动区按时间线展示该用户的下单、支付、分享、客服会话等行为，使转化路径全程可视。

## 五、触发事件

| 事件 | 触发时机 |
| --- | --- |
| `order_placed` | 用户在小程序商城提交订单时触发。 |
| `payment_success` | 用户完成支付、交易成功时触发。 |
| `customer_service_msg` | 用户在小程序内发送客服消息时触发。 |
| `share_clicked` | 用户点击分享或被分享链接被点击时触发。 |

## 七、关键指标

- 访问 UV/PV 与访问深度：衡量小程序的流量规模与浏览质量。
- 下单转化率与支付成功率：衡量从浏览到成交的转化效率。
- 客单价与复购率：衡量小程序的成交质量与客户价值。
- 分享率与裂变拉新数：衡量社交玩法带来的增量获客。
- 客服会话响应率与满意度：衡量小程序内服务体验。

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID / 事件总线）；微信/支付宝小程序官方 API 与支付能力用于商城、订阅消息、客服、授权、支付等对接。
- **边界**：本 Skill 负责小程序渠道的连接、交易/客服事件采集与消息下发，不负责商品内容与营销素材生产（交由内容模块），也不负责跨渠道自动化旅程编排与活动策略（交由活动/自动化模块）。
