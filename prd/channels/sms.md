# 短信 SMS

> **Skill ID** `sms-skill` ｜ **分类** 渠道 ｜ **文档版本** v0.2

## 一、模块定位

短信是覆盖最广、到达最稳的强触达通道，依托运营商基础网络可触达几乎所有持有手机号的用户，且打开率高达约 98%，远超其他营销渠道。正因如此，短信长期承担着验证码、交易/服务通知与营销推文等关键触达任务——尤其是验证码和重要通知，几乎不可被其他渠道替代。短信渠道 Skill（`sms-skill`）通过短信网关接入，把单发/群发、模板管理、回执追踪、定时发送、黑名单与频率控制等能力封装进 Nebula，使短信成为可控、可追踪、合规的统一触达通道之一。

本 Skill 解决短信触达"发得出去却看不见效果、管不好合规"的问题。Nebula 将短信的送达回执、失败、回复等结果实时接入事件总线，经 OneID 与客户主档绑定，使触达结果可回流分析并驱动后续动作；同时通过模板审批、黑名单与频率控制保障合规与体验，避免对用户的过度打扰。短信作为强触达兜底渠道，与微信生态、邮件等渠道协同，构成 Nebula 多渠道触达矩阵中"高到达率"的关键一环。

## 二、目标用户与典型场景

- **目标用户**：需要可靠下发验证码与交易通知的产品/技术运营、做促销与召回触达的营销运营、对触达合规有要求的会员/CRM 运营。
- **典型场景**：
  - 用户注册登录或交易关键环节下发验证码短信，依托高到达率保障流程顺畅。
  - 营销活动期间向目标人群定时群发促销短信，并通过回执追踪评估实际送达与效果。
  - 对退订或投诉用户加入黑名单并施加频率控制，避免重复打扰、保障合规触达。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 单发/群发短信 | 支持向单个用户或批量人群发送短信，覆盖验证码、通知与营销推文。 |
| 短信模板管理/审批 | 支持短信模板的创建与审批，保障内容合规并提升发送效率。 |
| 发送回执追踪 | 追踪每条短信的送达回执，让触达结果可见、可分析。 |
| 定时发送 | 支持按设定时间发送短信，匹配营销节奏与用户活跃时段。 |
| 黑名单管理 | 维护退订/投诉等黑名单，避免向不应触达的用户发送。 |
| 发送频率控制 | 限制对同一用户的发送频率，防止过度打扰、保障体验与合规。 |

## 四、ChatUI 交互

短信以下发为主，用户回复（如退订或咨询）经回执通道进入 ChatUI 中栏会话流，运营可在统一界面查看短信触达与回复情况；右栏客户卡片展示该用户绑定的手机号、短信触达历史与订阅/黑名单状态，并经 OneID 与其在其他渠道的画像融合，避免跨渠道的重复打扰。中栏 AI 建议区可基于客户画像与触达频率，提示是否适合发送、推荐模板或下一步跟进动作，运营一键采纳并触发发送。右栏活动区按时间线展示该用户的短信送达、失败、回复等事件，使强触达渠道的效果可视、可控。

## 五、触发事件

| 事件 | 触发时机 |
| --- | --- |
| `sms_delivered` | 短信成功送达用户、收到送达回执时触发。 |
| `sms_failed` | 短信发送失败、收到失败回执时触发。 |
| `sms_replied` | 用户对短信进行回复时触发。 |

## 七、关键指标

- 送达率：衡量短信实际到达用户的比例，是强触达渠道的核心指标。
- 发送失败率：衡量号码无效、网关异常等导致的触达损耗。
- 回复率：衡量用户对短信的响应程度（适用于互动类短信）。
- 退订/投诉率：衡量触达对用户体验的影响与合规风险。
- 营销短信转化率：衡量促销/召回短信带来的实际业务转化。

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID / 事件总线）；短信网关/运营商通道用于短信下发、模板报备与回执回传。
- **边界**：本 Skill 负责短信渠道的下发、回执与回复事件采集及合规控制，不负责短信文案的创作（交由内容模块），也不负责跨渠道自动化触达旅程的编排与策略设计（交由活动/自动化模块）。
