# 公众号 MP

> **Skill ID** `mp-skill` ｜ **分类** 渠道 ｜ **文档版本** v0.2

## 一、模块定位

微信公众号（含订阅号与服务号）是品牌在微信生态内进行内容分发与用户触达的核心阵地。订阅号侧重内容传播与品牌心智建设，服务号则凭借模板消息、菜单、卡券等能力承担服务通知与转化触达的职责。对大量中国品牌而言，公众号既是粉丝沉淀的"内容主页"，也是订单通知、活动提醒等关键服务消息的下发通道。公众号渠道 Skill（`mp-skill`）通过微信公众平台官方接口接入公众号，将模板消息、自定义菜单、图文推送、粉丝管理、智能回复、带参二维码与卡券等能力统一封装进 Nebula。

本 Skill 对标微盟、有赞等服务商的公众号运营模块，解决公众号运营中"内容、菜单、消息、粉丝数据各自为政"的问题。Nebula 把公众号能力作为标准技能接入统一数据层：粉丝的关注/取关、菜单点击、扫码来源等行为实时回流 CDP，与全域画像通过 OneID 打通；智能回复与 AI 客服由原生 AI Agent 驱动，使公众号从单向内容推送平台升级为可对话、可追踪、可转化的智能触点。

## 二、目标用户与典型场景

- **目标用户**：依托公众号做内容营销与会员服务的品牌新媒体/内容运营、用服务号下发交易与服务通知的电商运营、做活动拉新与渠道追踪的市场团队。
- **典型场景**：
  - 用户下单或订单状态变更时，通过服务号模板消息推送通知，并将触达与点击行为回流分析。
  - 在不同投放位放置带参二维码，用户扫码关注后自动识别渠道来源并归因，评估各渠道拉新效果。
  - 用户发送关键词触发智能回复或 AI 客服，自动解答常见问题并将会话沉淀为客户行为数据。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 模板消息 | 支持订单通知、活动提醒、状态变更等结构化消息的下发，承担服务号的关键触达职责。 |
| 自定义菜单 | 支持配置三级菜单并统计菜单点击事件，把菜单作为可分析的导航与转化入口。 |
| 图文推送 | 支持图文内容的编辑、发布、定时下发与按分组推送，实现精准的内容分发。 |
| 粉丝管理 | 管理关注/取关事件与粉丝标签，沉淀公众号的用户资产与分层基础。 |
| 智能回复 | 支持关键词自动回复与 AI 客服，自动响应粉丝咨询、降低人工接待成本。 |
| 二维码 | 支持带参二维码用于追踪渠道来源，实现拉新的精细化归因。 |
| 卡券 | 支持优惠券、会员卡等卡券的推送，连接内容触达与营销转化。 |

## 四、ChatUI 交互

粉丝在公众号内发来的消息（如咨询、关键词）实时进入 ChatUI 中栏的会话流，运营可在统一界面查看并处理，智能回复与 AI 客服自动兜底常见问题；右栏客户卡片展示该粉丝的公众号身份、关注状态、扫码来源与粉丝标签，并经 OneID 与其在小程序、企业微信等渠道的画像融合，呈现完整客户视图。中栏 AI 建议区基于会话与粉丝画像给出回复话术或可推送的卡券/图文建议，运营一键采纳。右栏活动区展示该粉丝的关注、菜单点击、扫码、消息触达等行为时间线，使内容运营效果一目了然。

## 五、触发事件

| 事件 | 触发时机 |
| --- | --- |
| `subscribe` | 用户关注公众号时触发。 |
| `unsubscribe` | 用户取消关注公众号时触发。 |
| `menu_click` | 用户点击自定义菜单项时触发。 |
| `qrcode_scanned` | 用户扫描带参二维码时触发。 |

## 七、关键指标

- 粉丝净增数与取关率：衡量公众号用户资产的增长与流失。
- 图文打开率与阅读完成率：衡量内容分发的吸引力。
- 模板消息送达率与点击率：衡量服务触达的有效性与转化引导。
- 菜单与二维码扫码量及渠道分布：衡量入口活跃度与拉新归因。
- 卡券领取与核销率：衡量内容触达向营销转化的转化效率。

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID / 事件总线）；微信公众平台官方 API 用于模板消息、菜单、图文、粉丝、二维码、卡券等能力对接。
- **边界**：本 Skill 负责公众号渠道的连接、消息下发与事件采集，不负责图文/营销内容的创作生产（交由内容模块），也不负责跨渠道自动化旅程的编排（交由活动/自动化模块）。
