# 企业微信 WeCom

> **Skill ID** `wecom-skill` ｜ **分类** 渠道 ｜ **文档版本** v0.2

## 一、模块定位

企业微信是腾讯面向企业推出的官方客户连接工具，也是当下中国 B2C 客户运营（SCRM）最核心的承载平台。相比个人微信，企业微信以企业主体对外，提供合规、可管理、可继承的客户关系底座，已成为品牌沉淀私域、规模化运营客户的事实标准。企业微信渠道 Skill（`wecom-skill`）通过官方接口对接企业微信，把客户联系、客户群、标签体系、会话存档等核心 SCRM 能力封装进 Nebula，让品牌在统一 ChatUI 中完成从加好友、打标签到群运营、合规存档的全链路私域操作。

本 Skill 对标尘锋、探马等专业 SCRM 产品，解决企业在企业微信上"工具割裂、数据孤岛"的痛点：传统做法需要在 SCRM 后台、侧边栏、群发助手等多个系统间反复切换，客户数据难以与品牌全域画像打通。Nebula 以原生 AI Agent 为内核，将企业微信能力作为可插拔技能接入统一数据层，使每一次客户互动、每一个标签变更都实时回流到 CDP 与 OneID，并由 AI 辅助生成回复与运营动作，把企业微信从"通讯工具"升级为"智能客户运营中枢"。

## 二、目标用户与典型场景

- **目标用户**：以企业微信为主阵地的 SCRM 运营团队、零售与品牌的私域/会员运营、需要合规留存客户沟通记录的金融/教育/医美等行业运营。
- **典型场景**：
  - 通过渠道活码从不同投放来源引流加好友，系统按来源自动打标签，实现精细化分层运营。
  - 运营借助侧边栏在接待客户时调出客户信息卡片与快捷话术，结合 AI 建议高效完成一对一服务。
  - 员工离职时，其名下客户关系按规则自动继承给接替同事，保障私域资产不流失。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 客户联系 | 支持一对一及群聊的客户连接，覆盖企业微信对外服务客户的基础通道。 |
| 客户群管理 | 支持客户群的创建、日常管理与裂变增长，把社群运营纳入统一管理。 |
| 标签体系 | 支持自动打标签与标签分组，构建结构化的客户分层与画像基础。 |
| 会话存档 | 合规保存员工与客户的聊天记录，满足监管与质检的留痕要求。 |
| 群发助手 | 支持批量向客户或客户群发送内容，实现规模化的私域触达。 |
| 渠道活码 | 不同投放渠道使用不同活码引流，加好友时自动按渠道来源打标签。 |
| 离职继承 | 员工离职时客户关系自动转移给接替人，保障客户资产连续性。 |
| 侧边栏 | 在企业微信会话侧边栏提供客服快捷回复与客户信息卡片，提升接待效率。 |

## 四、ChatUI 交互

客户从企业微信发来的消息实时汇入 ChatUI 中栏的会话流，运营在统一界面即可接待来自一对一会话与客户群的所有消息；右栏客户卡片展示该客户的企业微信身份、由渠道活码与标签体系自动沉淀的来源标签与分层标签，并经 OneID 融合呈现跨渠道的完整画像与历史行为。中栏的 AI 建议区结合会话上下文与客户标签，实时给出回复话术、推荐素材和下一步运营动作，运营一键采纳即可回复，与企业微信侧边栏的快捷能力形成呼应。右栏自动化区展示该客户的标签变更、入群、继承等关键事件时间线，使私域运营全程可视、可追溯。

## 五、触发事件

| 事件 | 触发时机 |
| --- | --- |
| `message_received` | 收到客户通过企业微信发来的消息时触发。 |
| `customer_added` | 客户添加成功、建立企业微信客户关系时触发。 |
| `group_joined` | 客户加入客户群时触发。 |
| `tag_changed` | 客户标签发生新增、变更或移除时触发。 |

## 七、关键指标

- 客户新增数与渠道来源分布：衡量各投放渠道的引流效果。
- 客户群规模与裂变率：衡量社群运营的扩张能力。
- 消息触达率与会话响应率：衡量私域触达与接待质量。
- 标签覆盖率：衡量客户分层与画像的完整度。
- 私域转化率与客户留存率：衡量企业微信运营的最终业务贡献。

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID / 事件总线）；企业微信官方 API 用于客户联系、客户群、标签、会话存档等能力对接。
- **边界**：本 Skill 负责企业微信渠道的连接、客户运营动作与事件采集，不负责营销内容生产（交由内容模块），也不负责跨渠道自动化旅程的编排与策略设计（交由活动/自动化模块）。
