# 钉钉 DingTalk

> **Skill ID** `dingtalk-skill` ｜ **分类** 渠道 ｜ **文档版本** v0.2

## 一、模块定位

钉钉是阿里巴巴旗下的企业协作平台，以考勤、审批、工作通知等高频办公场景为核心，在中小企业及传统行业中渗透率极高，是大量企业组织内外部沟通的默认工具。对面向企业客户的营销与服务团队而言，钉钉是触达企业组织、嵌入对方工作流的有效渠道。钉钉渠道 Skill（`dingtalk-skill`）通过钉钉开放平台接入，把工作通知、群机器人、审批、日历与通讯录等能力封装进 Nebula，使营销通知与协作动作能够直接落到企业用户的日常工作场景中。

本 Skill 与飞书 Skill 构成 Nebula 在企业级协作渠道上的"双渠道覆盖"——飞书侧重互联网与新经济企业，钉钉则覆盖广泛的中小企业与传统行业，两者互补以最大化触达不同类型的企业客户。Nebula 把钉钉能力作为标准技能接入统一数据层：工作通知的下发、群机器人消息、审批结果等通过事件总线回流，经 OneID 与客户主档关联；群机器人与通知由原生 AI Agent 辅助生成与编排，让营销推送、活动排期与审批协调都在钉钉内顺畅完成。

## 二、目标用户与典型场景

- **目标用户**：面向中小企业客户的 B2B 市场与销售团队、需要走预算/活动审批的营销负责人、用钉钉做内外部协同的运营团队。
- **典型场景**：
  - 通过工作通知向目标企业用户下发系统消息或营销推送，确保信息在工作场景内被高效触达。
  - 群机器人通过 Webhook 推送营销简报，并在被关键词命中时自动回复，承担轻量化的群内通知。
  - 营销活动与预算通过钉钉审批流转，审批完成后回流并触发活动排期与日历提醒。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 工作通知 | 支持系统消息与营销推送的下发，把营销信息直达企业用户的工作台。 |
| 群机器人 | 支持 Webhook 消息推送与关键词回复，承担群内的自动化通知与互动。 |
| 审批 | 支持营销活动与预算的审批流程，把审批环节嵌入营销执行链路。 |
| 日历 | 支持营销活动的排期与提醒，协调团队与客户的时间安排。 |
| 通讯录 | 同步企业组织架构，构建人与企业、客户的映射关系。 |

## 四、ChatUI 交互

客户或团队通过钉钉发来的消息、以及群机器人接收到的互动，实时进入 ChatUI 中栏会话流，运营在统一界面即可处理钉钉侧沟通；右栏客户卡片展示对方的钉钉身份与通讯录中的组织架构信息，并经 OneID 与客户主档及其他渠道画像关联，形成企业客户的完整视图。中栏 AI 建议区结合上下文给出回复话术、可推送的工作通知或可触发的审批/排期动作建议，运营一键采纳。右栏活动区展示该客户相关的工作通知触达、审批完成、群互动等事件时间线，使企业级触达过程清晰可追溯。

## 五、触发事件

| 事件 | 触发时机 |
| --- | --- |
| `message_received` | 收到客户或团队通过钉钉发来的消息时触发。 |
| `group_joined` | 用户加入相关群组时触发。 |
| `approval_completed` | 营销活动或预算审批完成时触发。 |

## 六、配置参数

```json
{"api_key":"your_dingtalk_api_key","webhook_url":"https://your-domain.com/webhook","enabled":true}
```

## 七、关键指标

- 工作通知送达率与已读率：衡量营销推送在工作场景内的触达效果。
- 群机器人响应率：衡量群内自动化互动的覆盖与及时性。
- 审批流转时长与通过率：衡量营销活动与预算审批的效率。
- 活动排期及时率：衡量日历协同的执行质量。
- 企业客户触达覆盖率：衡量钉钉渠道对目标企业的覆盖广度。

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID / 事件总线）；钉钉开放平台 API 用于工作通知、群机器人、审批、日历、通讯录等能力对接。
- **边界**：本 Skill 负责钉钉渠道的连接、消息与协作事件采集，不负责营销内容的策划创作（交由内容模块），也不负责跨渠道自动化旅程的编排与活动策略设计（交由活动/自动化模块）。
