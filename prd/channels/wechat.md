# 微信 WeChat

> **Skill ID** `wechat-skill` ｜ **分类** 渠道 ｜ **文档版本** v0.2

## 一、模块定位

微信是中国体量最大的社交平台，月活跃用户约 13 亿，几乎覆盖了所有具备消费能力的中国网民。个人微信不同于公众号或企业微信，它是品牌与消费者之间最贴近、最具私域属性的第一触点——一个好友关系往往意味着一次有温度、可持续的一对一沟通通道。微信渠道 Skill（`wechat-skill`）通过微信协议客户端对接个人微信账号，将原本散落在个人微信号里的好友消息、朋友圈互动、收付款行为统一接入 Nebula，使品牌能够在合规前提下把个人微信纳入可运营、可沉淀、可分析的私域资产体系。

本 Skill 解决的核心问题，是个人微信长期处于"人肉运营、数据黑盒"的状态：客服或导购用个人手机逐条回复，离职即流失，互动数据无法回流到 CDP。Nebula 把消息收发、好友管理、朋友圈互动与支付能力封装为标准化技能，让运营在统一的 ChatUI 中处理个人微信会话，并将每一次互动写入事件总线与客户主档。对标市面上的微信生态运营/SCRM 工具（如各类微信群控、私域 SOP 工具），Nebula 的差异在于以原生 AI Agent 驱动会话理解与建议，而非简单的脚本化群发。

## 二、目标用户与典型场景

- **目标用户**：以个人微信开展私域运营的 DTC/零售品牌导购与社群运营、依赖个人号维系客户的本地生活与服务行业、需要把个人微信触点纳入统一客户视图的私域运营团队。
- **典型场景**：
  - 导购通过个人微信接待咨询客户，AI 自动识别意图并给出话术建议，运营一键回复并完成成交。
  - 品牌运营在朋友圈定时发布新品内容，系统自动捕捉客户的点赞与评论互动，回流为兴趣标签用于后续触达。
  - 客户通过微信收款码或转账完成支付，支付事件实时写入客户主档，触发后续的复购关怀流程。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 消息收发 | 支持文本、图片、语音、视频、链接等多种消息类型的双向收发，覆盖个人微信日常沟通的完整内容形态。 |
| 好友管理 | 支持好友的添加、删除与备注/标签维护，把零散的好友关系结构化为可分组、可检索的客户名册。 |
| 朋友圈 | 支持发布品牌内容到朋友圈，并查看客户对内容的点赞、评论等互动，将内容曝光转化为可量化的兴趣信号。 |
| 支付 | 支持收款码、红包、转账等微信支付场景，把交易行为接入数据层用于成交与复购分析。 |
| 群聊管理 | 支持品牌微信群的日常运营，统一管理群内消息与成员，沉淀社群互动数据。 |
| 朋友圈广告 | 支持朋友圈广告的定向投放，面向目标人群进行内容曝光与拉新引流。 |

## 四、ChatUI 交互

当客户从个人微信发来消息时，消息实时进入 ChatUI 中栏的会话流，运营无需切换到手机微信即可在统一界面里处理所有个人微信会话；右栏客户卡片展示该客户的微信身份（昵称、备注、头像）以及已沉淀的好友标签和分组，并叠加来自其他渠道经 OneID 归并后的完整画像，让运营在回复前就掌握客户全貌。中栏下方的 AI 建议区会基于会话上下文实时给出意图判断与话术草稿、推荐素材或下一步动作，运营确认后一键发送，既保留人工把关也大幅提升响应效率。右栏的活动与自动化区同步展示该客户的朋友圈互动、支付记录等行为时间线。

## 五、触发事件

| 事件 | 触发时机 |
| --- | --- |
| `message_received` | 收到客户通过个人微信发来的任意类型消息时触发。 |
| `friend_added` | 新好友添加成功、建立好友关系时触发。 |
| `moments_interacted` | 客户对品牌朋友圈内容产生点赞或评论等互动时触发。 |
| `payment_received` | 通过收款码、红包或转账收到客户付款时触发。 |

## 六、配置参数

```json
{"api_key":"your_wechat_api_key","webhook_url":"https://your-domain.com/webhook","enabled":true}
```

## 七、关键指标

- 消息响应率与平均首响时长：衡量个人微信会话的接待及时性。
- 好友新增数与好友留存率：衡量私域关系的扩张与稳固程度。
- 朋友圈互动率：衡量品牌内容在好友圈层的渗透与吸引力。
- 支付转化率与客单价：衡量个人微信触点的直接成交贡献。
- 客户活跃度：衡量好友的持续互动与私域资产健康度。

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID / 事件总线）；微信协议客户端用于对接个人微信账号、收发消息与采集互动数据。
- **边界**：本 Skill 只负责个人微信渠道的连接、消息收发与事件采集，不负责营销内容的策划与生产（交由内容模块），也不负责跨渠道的自动化旅程编排（交由活动/自动化模块）。
