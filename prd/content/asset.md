# 素材库 Assets

> **Skill ID** `asset-skill` ｜ **分类** 内容 ｜ **文档版本** v0.2

## 一、模块定位

素材库是 Nebula 星云的企业营销内容统一仓库，集中管理图片、视频、文件、链接、音频等各类营销素材。它把企业在各类活动、各个渠道中产生的富媒体资产收口到一个可检索、可打标、可审核的中心化仓库，避免素材散落在个人电脑、聊天记录与各渠道后台中难以复用。素材库与模板库互为补充：模板管理文字骨架，素材管理富媒体血肉，二者共同构成内容生产的物料底座。

模块解决的核心问题是营销素材的"找不到、用不对、管不住"：找不到——素材分散无统一检索；用不对——缺乏用途/渠道标签导致错用；管不住——无版本与审核流程，合规风险高。素材库通过标签体系、全文检索、版本管理与合规审核流程系统化应对。对标产品上，它借鉴有赞素材中心的渠道化管理、微盟营销素材库的活动维度组织，以及飞书文档素材管理的检索与协作体验。

## 二、目标用户与典型场景

- **目标用户**：内容运营、设计与创意人员、市场活动负责人，以及需要合规把关的审核角色
- **典型场景**：
  - 活动物料归集：市场为一场大促上传海报、短视频与落地页链接，按"活动+渠道"打标签，团队成员随用随取。
  - 对话中快速发图：客服或运营在 ChatUI 对话中需要发送产品图，直接从素材面板拖拽素材进回复框即可发送。
  - 合规审核：新素材发布前进入审核流程，由合规角色检查后才允许在渠道中被引用，规避违规风险。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 多类型上传 | 支持图片、视频、PDF、链接、音频等多种素材类型上传，覆盖营销场景的全部物料形态。 |
| 标签体系 | 按用途、渠道、活动等多维度为素材打标签，形成结构化分类，便于精准筛选与跨场景复用。 |
| 全文检索 | 支持对文件名、标签与描述进行全文搜索，快速定位目标素材，降低查找成本。 |
| 版本管理 | 记录素材的更新历史，支持查看与追溯不同版本，避免误用过期物料。 |
| 合规审核 | 提供发布前审核流程，素材需通过审核才能进入可用状态，保障内容合规与品牌一致。 |
| ChatUI 拖拽发送 | 在对话界面提供素材面板，可将素材直接拖入回复框完成发送，打通"管理即使用"的最后一步。 |

## 四、ChatUI 交互

素材库在三栏 ChatUI 中以"素材面板"形态呈现，通常作为中栏对话区旁的可调用资源。运营在与客户或 AI 对话时，可唤起素材面板，按标签筛选或全文检索后，将目标图片、视频或链接直接拖拽进回复框，即完成内容引用与发送，无需跳转到独立后台。AI 也会基于对话上下文，在中栏以内容卡片形式推荐契合当前场景的素材。左栏导航沉淀素材分类与审核状态入口，右栏客户卡片为素材推荐提供画像依据，使素材选择更贴合当前客户。

## 五、与其他模块的关系

素材库通常不直接发出渠道事件，而是作为富媒体内容的供给方被消费：

- **被渠道模块发送**：各渠道在下发图文/视频消息时引用素材库中的资产 URL。
- **被模板库引用**：模板中的富媒体位可绑定素材库素材，实现图文一体的消息。
- **为个性化提供内容池**：个性化模块从素材库自动提取与客户偏好匹配的内容进行推荐。
- **被内容中台统一调度**：内容中台在分发排期时从素材库取物料并分配到渠道与时间。

## 六、配置与数据结构

素材的关键属性围绕类型、标签、版本与审核状态展开，示意结构如下：

```json
{
  "assetId": "ast_20260601_001",
  "name": "夏季大促主视觉",
  "type": "image",
  "url": "https://cdn.nebula.example/ast/summer.jpg",
  "tags": {
    "purpose": ["promotion"],
    "channel": ["wechat", "email"],
    "campaign": ["summer_sale_2026"]
  },
  "description": "夏季大促首页 banner，1080x1920",
  "version": 2,
  "auditStatus": "approved"
}
```

- **type**：素材类型（图片/视频/PDF/链接/音频）。
- **tags**：用途/渠道/活动多维标签，支撑全文检索与筛选。
- **auditStatus**：审核状态，未通过的素材不可被渠道引用。

## 七、关键指标

- 素材复用率（单素材被多渠道/多活动引用次数）
- 检索命中率与平均查找耗时
- 审核通过率与平均审核时长
- 素材库总量与活跃素材占比
- 拖拽发送使用频次

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID）提供客户画像以支撑素材推荐；对象存储/CDN 承载素材文件；个性化与内容中台模块作为主要消费方。
- **边界**：不负责文字模板的变量与版本管理（由模板库负责），不负责消息的实际发送（由渠道模块负责），不生成原创创意内容（由内容中台的 AI 辅助生产承担）。
