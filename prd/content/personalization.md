# 个性化 Personalization

> **Skill ID** `personalization-skill` ｜ **分类** 内容 ｜ **文档版本** v0.2

## 一、模块定位

个性化模块是基于客户画像与行为的智能推荐引擎，能够从素材库自动匹配每位客户偏好的内容，实现"千人千面"的内容触达。它把统一数据层沉淀的画像与行为信号转化为可执行的内容推荐，让营销从"群发同一套内容"升级为"为每个人挑最合适的内容"。作为内容体系中的智能匹配层，它连接素材库（内容供给）与渠道/活动（触达执行），是提升内容相关性与转化的关键引擎。

模块解决的核心问题是内容与人的错配：企业素材丰富，但若无差别群发，相关性低、打扰感强、转化差。个性化通过多维画像、组合推荐策略与跨渠道偏好统一，把对的内容在对的渠道推给对的人。对标产品上，它借鉴 Marketo Prediction 的预测式内容推荐与有赞个性化推荐的策略组合，强调规则引擎、协同过滤与实时更新的融合。

## 二、目标用户与典型场景

- **目标用户**：增长运营、CRM/会员运营、内容运营，以及关注复购与留存的精细化运营团队
- **典型场景**：
  - 偏好内容推送：基于客户的浏览与购买行为，从素材库自动匹配其偏好的品类内容进行推送，提升点击与转化。
  - 跨渠道偏好一致：同一客户在微信、短信、邮件等渠道收到风格与品类一致的推荐，避免割裂体验。
  - 实时响应行为：客户刚浏览某商品后，推荐策略实时更新，触达内容随之调整，抓住即时兴趣窗口。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 画像维度 | 融合人口属性、行为属性与心理属性三类画像维度，构建立体的客户理解，作为推荐的输入基础。 |
| 推荐策略 | 组合规则引擎、协同过滤与实时更新等策略，兼顾业务规则的可控性与算法的泛化能力。 |
| 跨渠道统一 | 同一客户在各渠道的偏好保持同步，保证多触点体验的一致性，避免重复或矛盾推荐。 |
| 内容匹配 | 从素材库自动提取与客户偏好匹配的内容，把抽象的偏好转化为具体可发送的物料。 |
| 效果追踪 | 跟踪推荐命中率与转化率，持续评估并优化推荐质量，形成数据闭环。 |

## 四、ChatUI 交互

个性化在三栏 ChatUI 中以"智能内容卡片"形态呈现。当运营与某位客户在中栏对话，或在配置触达流程时，右栏客户卡片展示该客户的画像维度与行为洞察，AI 据此在中栏主动推荐契合其偏好的内容卡片——运营可一键采用并发送，也可在卡片内微调。推荐内容直接取自素材库，做到"看到客户画像，就看到该发什么"。左栏导航沉淀推荐策略与效果追踪入口，整个推荐到发送的过程在对话中顺滑完成，体现 ChatUI 统一交互与画像驱动的产品理念。

## 五、与其他模块的关系

个性化本身不直接发出渠道事件，而是产出推荐结果供其他模块消费：

- **从素材库取内容**：推荐结果的具体物料从素材库自动提取匹配。
- **从数据层取画像**：人口/行为/心理画像与实时行为信号来自统一数据层（CDP / OneID）。
- **被活动/渠道模块消费**：自动化、触发器在编排触达时调用个性化推荐，由渠道完成实际发送。
- **与 A/B 测试协同**：推荐策略可通过 A/B 测试验证，效果回流持续优化策略权重。

## 六、配置与数据结构

个性化配置围绕画像维度、推荐策略与匹配规则展开，示意结构如下：

```json
{
  "strategyId": "psn_default",
  "profileDimensions": ["demographic", "behavioral", "psychographic"],
  "recommendStrategy": {
    "ruleEngine": true,
    "collaborativeFiltering": true,
    "realtimeUpdate": true
  },
  "contentSource": "asset-library",
  "crossChannelSync": true,
  "tracking": ["hit_rate", "conversion_rate"]
}
```

- **profileDimensions**：画像维度（人口/行为/心理属性）。
- **recommendStrategy**：推荐策略组合（规则引擎 + 协同过滤 + 实时更新）。
- **crossChannelSync**：是否跨渠道同步偏好。

## 七、关键指标

- 推荐命中率（被点击/采纳的推荐占比）
- 推荐转化率与带来的 GMV 增量
- 跨渠道推荐一致性覆盖率
- 实时推荐响应延迟
- 画像覆盖完整度（具备可用画像的客户占比）

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID）提供画像与实时行为；素材库提供可匹配的内容池；渠道/活动模块作为推荐结果的执行方；A/B 测试用于策略验证。
- **边界**：不负责内容的生产与存储（由素材库/模板库负责），不负责消息的实际发送（由渠道模块负责），不负责触达时机的流程编排（由活动模块负责）。
