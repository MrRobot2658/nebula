# 内容中台 Content Hub

> **Skill ID** `content-hub-skill` ｜ **分类** 内容 ｜ **文档版本** v0.2

## 一、模块定位

内容中台是企业级的内容管理中枢，统一承载内容的生产、审核、分发与分析全生命周期。它把模板库、素材库、个性化、A/B 测试等单点内容能力编排为一条完整的内容运营流水线，适合需要管理大量营销内容、多团队协作、多渠道排期的中大型企业。如果说模板库与素材库是"物料仓库"，内容中台就是统筹物料从生产到上线再到复盘的"内容指挥中心"。

模块解决的核心问题是规模化内容运营的协同与治理：内容多、人多、渠道多时，谁在生产、是否审核、何时分发到哪个渠道、表现如何，往往缺乏统一视图与流程管控，导致重复生产、合规漏洞与排期混乱。内容中台通过 AI 辅助生产、多级审核、分发排期、效果分析、内容日历与知识库沉淀，把这些环节串成可视、可控、可复用的闭环。理念上借鉴致趣百川的内容营销方法论与有赞内容运营体系，强调"内容资产化、运营流程化"。

## 二、目标用户与典型场景

- **目标用户**：内容运营负责人、市场内容团队、合规审核人员，以及中大型企业的内容运营管理者
- **典型场景**：
  - 批量内容生产：借助 AI 辅助写作与批量生成，快速产出多渠道、多版本的营销文案初稿，再由人工精修。
  - 多级审核分发：内容经过多级审核与合规检查后，按渠道与时间排期分发，确保上线内容合规且有序。
  - 内容排期复盘：通过内容日历可视化管理排期，活动结束后对比各渠道内容表现，沉淀有效经验到知识库。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 内容生产 | 提供 AI 辅助写作与批量生成能力，加速多渠道、多版本内容的初稿产出，降低规模化生产门槛。 |
| 审核流程 | 支持多级审核与合规检查，内容上线前层层把关，规避品牌与合规风险。 |
| 分发管理 | 统筹"素材→渠道→时间排期"的分发链路，让内容有计划、有节奏地触达。 |
| 效果分析 | 对比各渠道内容的表现数据，识别高/低效内容，指导后续生产与投放决策。 |
| 内容日历 | 以可视化日历呈现内容排期，全局掌握各渠道、各时间点的内容安排，避免冲突与空档。 |
| 知识库 | 沉淀营销知识、优秀案例与可复用方法论，支撑内容资产的长期积累与团队复用。 |

## 四、ChatUI 交互

内容中台在三栏 ChatUI 中作为内容运营的指挥界面呈现。运营在中栏可通过自然语言驱动 AI 辅助写作与批量生成（如"为夏季大促生成三版微信文案"），AI 以内容卡片返回草稿，运营在对话中精修并提交审核流程。中栏的内容卡片可串联审核状态、分发排期与效果数据，形成从生产到复盘的一站式视图；内容日历则以可视化形式嵌入，便于全局排期。左栏导航沉淀审核流程、内容日历与知识库入口，右栏可结合客户分群辅助内容定向。整个内容生命周期都在 ChatUI 统一交互中被编排与监控。

## 五、与其他模块的关系

内容中台是内容能力的编排者，通常不直接发出渠道事件，而是统筹各内容模块协同：

- **统筹模板库与素材库**：在生产与分发环节调用模板与素材作为物料来源。
- **调度个性化与 A/B 测试**：分发时结合个性化推荐，并可对内容发起 A/B 测试择优。
- **由渠道模块执行下发**：分发排期最终由各渠道模块在指定时间触达。
- **效果数据回流**：各渠道内容表现回流到效果分析与知识库，形成内容资产闭环。

## 六、配置与数据结构

内容中台的关键结构围绕生产、审核、分发与排期展开，示意结构如下：

```json
{
  "contentId": "hub_summer_001",
  "title": "夏季大促主推文",
  "production": { "aiAssisted": true, "batchGenerated": true },
  "auditFlow": [
    { "level": 1, "role": "editor", "status": "approved" },
    { "level": 2, "role": "compliance", "status": "pending" }
  ],
  "distribution": {
    "assets": ["ast_20260601_001"],
    "channels": ["wechat", "email"],
    "scheduleAt": "2026-07-01T10:00:00"
  },
  "calendarSlot": "2026-W27",
  "knowledgeTags": ["promotion", "summer"]
}
```

- **auditFlow**：多级审核链路与合规检查状态。
- **distribution**：素材→渠道→时间排期的分发配置。
- **calendarSlot / knowledgeTags**：内容日历排期位与知识库沉淀标签。

## 七、关键指标

- 内容产能（单位时间产出内容量）与 AI 辅助占比
- 审核通过率与平均审核时长
- 排期准时分发率
- 各渠道内容表现对比（打开/点击/转化）
- 知识库复用率与高效内容沉淀数量

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID）提供受众与效果数据；模板库、素材库提供生产物料；个性化与 A/B 测试参与分发优化；渠道模块执行最终下发。
- **边界**：不负责单条消息的实际发送与渠道适配（由渠道模块负责），不替代单点内容能力本身（模板/素材/个性化/A B 测试各司其职），不负责客户数据的采集与建模（由统一数据层负责）。
