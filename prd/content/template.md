# 模板库 Templates

> **Skill ID** `template-skill` ｜ **分类** 内容 ｜ **文档版本** v0.2

## 一、模块定位

模板库是 Nebula 星云内容生产体系的基础设施，负责统一管理跨渠道的消息模板。无论是微信、短信、邮件、企业微信，还是其余渠道，所有出站消息的"骨架"都沉淀在这里。它把原本散落在各渠道、各运营手上的零散文案，收敛为一套可复用、可版本化、可变量化的标准模板资产，让"写一次、处处适配"成为可能。模板库不直接发送消息，而是作为渠道与活动模块调用的内容来源，承担"模板即代码"式的集中治理。

模块解决的核心问题是内容生产的重复劳动与一致性失控：运营在不同渠道反复手写相似文案，品牌话术难以统一，促销规则一改就要逐条修改散落的文案。模板库通过分类、变量注入、多版本与历史回溯，把这些痛点系统化解决。对标产品上，它借鉴了微盟素材中心的集中管理思路与有赞模板消息的渠道化下发能力，并在此基础上强化了 ChatUI 实时预览与多渠道自动适配。

## 二、目标用户与典型场景

- **目标用户**：内容运营、市场文案、渠道运营，以及需要维护标准化话术的客服与增长团队
- **典型场景**：
  - 新客欢迎：运营创建一条"欢迎"分类模板，注入 `{{customer.name}}` 等变量，配置后由各渠道在新客注册时自动调用并适配为对应格式。
  - 促销活动：市场团队建立"促销"模板，使用 `{{campaign.discount}}` 动态填充不同活动的折扣，活动一变只需改模板，无需逐条改文案。
  - A/B 文案迭代：同一条提醒模板维护多个版本，配合 A/B 测试挑选最优表达，胜出版本一键设为主版本并保留历史回滚能力。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 模板分类 | 按业务用途划分为欢迎、促销、提醒、确认、通知等类别，便于运营快速检索与管理，并支撑渠道按场景调用正确的模板。 |
| 变量注入 | 支持 `{{customer.name}}`、`{{campaign.discount}}` 等占位符，发送时从 CDP/活动上下文动态替换为真实值，实现千人千面的个性化文案。 |
| 多版本管理 | 同一模板可维护多个版本，天然支撑 A/B 测试与灰度切换，便于对比不同表达的转化效果。 |
| ChatUI 实时预览 | 在中栏对话区编辑模板时即时渲染最终效果，变量以示例值填充，所见即所发，降低上线前的检查成本。 |
| 多渠道适配 | 同一模板自动适配微信、短信、邮件、企微等不同渠道的格式与长度约束，无需为每个渠道重复维护内容。 |
| 历史版本追溯与回滚 | 完整记录每次修改的版本快照，可追溯改动并一键回滚到任意历史版本，保障内容安全与可审计。 |

## 四、ChatUI 交互

模板库深度嵌入三栏式 ChatUI。运营在中栏对话区通过自然语言唤起模板技能（如"帮我建一条欢迎模板"），AI 以内容卡片形式给出模板草稿与推荐变量，运营可在卡片内直接编辑并看到实时预览——变量 `{{customer.name}}` 即时以示例客户名渲染，效果与最终发送完全一致。左栏导航沉淀模板分类与历史版本入口，右栏客户卡片提供变量可取用的真实字段参考。模板确认后即可被回复框或活动流程引用，实现"边聊边建模板、即建即用"的顺滑体验。

## 五、与其他模块的关系

模板库通常不直接发出渠道事件，而是作为被动的内容供给方被消费：

- **被渠道模块发送**：微信、短信、邮件、企微等渠道模块在发消息时引用模板 ID，由渠道完成变量替换与格式适配后真正下发。
- **被活动模块编排**：自动化、触发器等活动模块在流程节点中绑定模板，按时机调用。
- **向 A/B 测试供给版本**：多版本能力为 A/B 测试模块提供可对比的内容变体，测试结果回流后指导主版本选择。
- **从数据层取变量值**：变量替换所需的真实字段来自统一数据层（CDP / OneID）。

## 六、配置与数据结构

模板的关键配置围绕分类、变量与版本展开，示意结构如下：

```json
{
  "templateId": "tpl_welcome_001",
  "name": "新客欢迎",
  "category": "welcome",
  "channels": ["wechat", "sms", "email", "wecom"],
  "currentVersion": "v3",
  "versions": [
    {
      "version": "v3",
      "content": "欢迎光临 {{customer.name}}！我们为您准备了 {{campaign.discount}} 优惠。",
      "variables": ["customer.name", "campaign.discount"],
      "updatedAt": "2026-06-20"
    }
  ]
}
```

- **category**：模板分类（欢迎/促销/提醒/确认/通知）。
- **variables**：变量列表，发送时从 CDP/活动上下文解析。
- **versions**：版本数组，支撑多版本管理与历史回滚。

## 七、关键指标

- 模板复用率（一条模板被多渠道/多活动调用的次数）
- 变量替换成功率（避免出现空值或占位符泄漏）
- 模板下发量与覆盖渠道数
- 版本迭代频次与回滚次数
- 模板维护工时节省（相较散点手写文案）

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID）提供变量替换所需的客户与活动字段；A/B 测试模块协同进行版本择优；渠道模块完成最终适配与下发。
- **边界**：不负责消息的实际发送与渠道协议适配（由渠道模块负责），不负责发送时机编排（由活动模块负责），也不直接管理图片视频等富媒体素材（由素材库负责）。
