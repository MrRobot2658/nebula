# A/B 测试 A/B Testing

> **Skill ID** `abtest-skill` ｜ **分类** 内容 ｜ **文档版本** v0.2

## 一、模块定位

A/B 测试模块负责对消息内容、发送时间、渠道组合等营销变量进行分流实验，并基于统计显著性自动选出胜出方案。它把"凭经验拍脑袋"的内容决策升级为"数据驱动的科学实验"，让运营在小流量上验证假设、在大流量上放大确定性收益。作为内容体系中的优化引擎，它依赖模板库提供可对比的内容版本，并将实验结论回流到分析与内容决策中。

模块解决的核心问题是营销决策缺乏量化依据：同样一条促销文案，换个标题、换个 CTA、换个发送时间，转化可能相差数倍，但人工难以判断孰优孰劣，也难以判断差异是否显著（而非随机波动）。A/B 测试通过受控分流、显著性判定与自动择优，把这种不确定性转化为可复用的最优实践。对标产品上，它借鉴 Marketo A/B Testing 的营销实验框架与 Google Optimize 的统计显著性方法论。

## 二、目标用户与典型场景

- **目标用户**：增长运营、内容运营、市场策略人员，以及关注 ROI 的活动负责人
- **典型场景**：
  - 文案择优：对同一促销活动准备两版标题与 CTA，以 50/50 分流测试，系统自动判定显著性后推送胜出版本给剩余人群。
  - 发送时间优化：测试上午与晚间两个发送时间对打开率的影响，找到最佳触达时机。
  - 渠道组合实验：对比"短信单触达"与"短信+企微双触达"的转化率与 ROI，确定性价比最优的渠道策略。

## 三、核心能力

| 能力 | 说明 |
| --- | --- |
| 可测变量 | 支持对内容、标题、CTA、发送时间、渠道等多类变量设计实验，覆盖营销触达的关键决策点。 |
| 分流规则 | 支持 50/50、70/30 以及多组分流等多种比例配置，灵活适配验证型与放量型实验。 |
| 多维指标 | 以打开率、点击率、转化率、ROI 等核心指标衡量各组表现，全面评估方案优劣。 |
| 统计显著性判定 | 自动计算各组差异是否达到统计显著，避免被随机波动误导，确保结论可靠。 |
| 自动择优推送 | 在显著性达成后自动选出胜出方案，并向剩余受众推送，最大化整体收益。 |
| 可视化测试报告 | 以图表形式对比各组指标与置信度，沉淀可复用的实验结论。 |

## 四、ChatUI 交互

A/B 测试在三栏 ChatUI 中以对话式实验配置呈现。运营在中栏通过自然语言描述实验意图（如"测一下这两版标题哪个点击率高，50/50 分流"），AI 以内容卡片形式生成实验配置——可测变量、分流比例、目标指标一目了然，运营确认即可启动。实验进行中，中栏以卡片实时更新各组指标与显著性进度；达成显著后，AI 主动提示胜出方案并询问是否放量。左栏导航沉淀实验列表与历史报告，右栏可结合客户分群信息辅助受众选择，使实验配置与结果消费都在对话中完成闭环。

## 五、与其他模块的关系

A/B 测试本身不直接发出渠道事件，而是编排内容版本并消费数据：

- **从模板库取版本**：实验所对比的内容变体来自模板库的多版本能力。
- **由渠道模块执行触达**：分流后的各组消息实际发送由渠道模块完成。
- **结果回流分析**：实验各组的打开/点击/转化数据回流到分析模块与统一数据层，形成可追溯的实验资产。
- **指导内容决策**：胜出方案反哺模板库主版本选择与个性化推荐策略。

## 六、配置与数据结构

实验配置围绕变量、分流与指标展开，示意结构如下：

```json
{
  "testId": "abt_summer_title_001",
  "variable": "title",
  "groups": [
    { "name": "A", "ratio": 0.5, "templateVersion": "v1" },
    { "name": "B", "ratio": 0.5, "templateVersion": "v2" }
  ],
  "metrics": ["open_rate", "click_rate", "conversion_rate", "roi"],
  "significanceThreshold": 0.95,
  "autoPromoteWinner": true
}
```

- **variable**：被测变量（内容/标题/CTA/发送时间/渠道）。
- **groups.ratio**：各组分流比例，支持 50/50、70/30、多组。
- **significanceThreshold**：统计显著性阈值，达成后触发自动择优。

## 七、关键指标

- 各组打开率、点击率、转化率与 ROI 对比
- 实验达成统计显著的比例与平均所需样本量
- 胜出方案相对基线的提升幅度（lift）
- 实验上线到出结论的平均周期
- 自动择优放量带来的增量收益

## 八、依赖与边界

- **依赖**：统一数据层（CDP / OneID）提供受众与行为指标；模板库提供内容版本；渠道模块执行各组触达；分析模块承接结果回流。
- **边界**：不负责内容本身的创作与版本存储（由模板库/素材库负责），不负责消息发送的渠道适配（由渠道模块负责），不替代长期归因分析（由分析模块负责）。
