# 统一数据层 Unified Data Layer

> **文档版本** v0.2

统一数据层是所有 Skill 共享的数据基础设施，由 **CDP + OneID + 事件总线** 三部分组成。它让全渠道的客户、行为与事件统一可用，是 Nebula「全域营销」的底座。

## 一、CDP（客户数据平台）

CDP 集成所有渠道的客户数据，对标创略科技 CDP 和销售易 CDP 的能力。数据按四类维度统一建模：

| 数据类型 | 内容 |
| --- | --- |
| **行为数据** | 浏览、点击、购买、退款、客服咨询 |
| **属性数据** | 年龄、地域、行业、职级、企业信息 |
| **互动数据** | 咨询、投诉、好评、会议参与 |
| **交易数据** | 订单金额、频次、品类偏好、客单价 |

这四类数据共同构成客户画像，供个性化、评分、分析等模块调用。

## 二、OneID（身份统一）

- **ID 映射**：微信 OpenID / 企微 ExternalUserID / 小程序 UnionID / 手机号 / 邮箱 / 设备 ID
- **合并策略**：确定性匹配（手机号 / UnionID）+ 概率匹配（行为相似度）
- **冲突解决**：优先级规则 + 人工合并

OneID 确保同一客户在微信、小程序、企微、官网等多渠道的身份被自动识别并合并为一份档案。

## 三、事件总线

- 发布 / 订阅模式，**至少一次投递**
- 事件**持久化**至数据库，支持**回放**
- 跨 Skill 消息延迟 **< 100ms**

事件总线把渠道侧产生的事件实时分发给内容与活动模块，是自动化、触发器、评分、分析得以实时运转的前提。
