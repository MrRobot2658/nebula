# 线上会议 Webinar

> **Skill ID** `webinar-skill` ｜ **分类** 活动 ｜ **文档版本** v0.2

## 一、模块定位
线上会议负责线上会议、直播、研讨会的全流程管理，把会议创建、报名、提醒、参与追踪与会后跟进整合为一条贯通的营销链路。它既覆盖 B2B 场景下的研讨会/网络直播获客，也支持电商场景的直播带货，是 Nebula 把"一场线上活动"转化为可追踪、可运营线索资产的核心活动模块。

该模块解决的核心问题是线上活动的散点化运营：会议工具、报名表单、提醒推送、参会数据、会后跟进往往割裂在不同系统，数据无法回流到客户画像。线上会议把全流程打通，让每位参会者的报名、出席、互动、转化行为都沉淀为事件，回流至 CDP 并驱动后续自动化。对标 Zoom Webinar + Marketo Event Integration，并融合有赞直播、微盟直播的电商直播能力。

## 二、目标用户与典型场景
- **目标用户**：市场活动经理、内容/品牌运营、直播运营、增长运营（负责线上获客与转化的角色）
- **典型场景**：
  - B2B 研讨会获客：发布报名页收集线索，会前多渠道提醒提升出席，会中问答投票互动，会后自动发回放并按参与度分级跟进。
  - 电商直播带货：直播间内商品上架、实时下单，结合限时权益促成即时转化。
  - 会后线索激活：根据"是否全程参与/中途退出"自动打分打标签，把高参与度听众推入销售跟进流程。

## 三、核心能力
| 能力 | 说明 |
| --- | --- |
| 会议创建 | 配置标题、时间、嘉宾、议程等基础信息，快速搭建一场线上会议/直播/研讨会。 |
| 报名管理 | 提供报名表单、报名审核与确认通知，管理报名名单与准入资格。 |
| 提醒推送 | 会前 N 分钟通过微信/短信/邮件多渠道提醒，降低爽约率、提升出席率。 |
| 互动工具 | 内置问答、投票、问卷、资料下载等互动工具，提升参与感并收集意向信号。 |
| 参与追踪 | 追踪谁来了、谁全程参与、谁中途退出，量化每位参会者的实际投入度。 |
| 会后跟进 | 自动发送回放、资料与满意度调查，把会议热度延续到会后转化。 |
| 数据报表 | 统计报名率、出席率、互动率、转化率，全面评估会议效果。 |
| 电商直播 | 支持商品上架、直播带货与实时下单，打通内容种草到即时成交的链路。 |

## 四、ChatUI 交互
运营可在对话栏说"建一场下周三的产品发布直播，会前 30 分钟微信+短信提醒"，Agent 据此生成会议草稿与提醒计划，并在对话中给出报名页与提醒文案的 AI 建议。会议创建表单、议程编排、报名审核、互动工具配置等以右栏面板或独立页呈现；直播进行中，报名/出席/互动的实时数据以状态卡片在对话上下文中滚动可见。右栏客户卡片在查看某客户时，可联动显示其在历次会议中的报名与出席记录，使线上活动数据成为客户上下文的一部分，贯通三栏。

## 五、事件与编排
线上会议既向事件总线产出标准事件，也消费事件驱动提醒与跟进。

| 事件 | 触发的编排 |
| --- | --- |
| `webinar.register`（报名） | 发送确认通知、加入会前提醒序列 |
| `webinar.remind`（会前定时） | 多渠道提醒推送（微信/短信/邮件） |
| `webinar.join` / `webinar.leave`（入会/退出） | 记录出席与时长，更新参与度 |
| `webinar.interact`（问答/投票/下载） | 累加评分、打意向标签 |
| `webinar.end`（结束） | 自动发回放、资料、满意度调查 |

这些事件回流事件总线后，可被评分模型加分、被触发器命中、被自动化流程作为入口，把会议参与度转化为后续营销动作。

## 六、配置与数据结构
会议配置涵盖基础信息、报名、提醒与互动设置：

```json
{
  "webinarId": "web_launch_0701",
  "title": "产品发布直播",
  "startTime": "2026-07-01T20:00:00+08:00",
  "hosts": ["嘉宾A", "嘉宾B"],
  "agenda": ["开场", "新品演示", "Q&A"],
  "registration": { "form": true, "audit": false, "confirmNotify": true },
  "reminders": [
    { "beforeMin": 60, "channels": ["wechat", "sms"] },
    { "beforeMin": 5, "channels": ["wechat"] }
  ],
  "interactions": ["qa", "poll", "survey", "download"],
  "commerce": { "liveSelling": true, "products": ["sku_001", "sku_002"] }
}
```

## 七、关键指标
- 报名率 / 出席率（实到/报名）
- 全程参与率与平均观看时长
- 互动率（问答/投票/下载参与度）
- 会后转化率与直播 GMV
- 满意度调查得分

## 八、依赖与边界
- **依赖**：统一数据层（CDP / OneID / 事件总线）；渠道与内容模块（提醒推送依赖微信/短信/邮件渠道与内容模板）；评分模型、触发器、自动化流程（消费会议事件做后续编排）。
- **边界**：不自研音视频会议底层能力（对接 Zoom/直播服务），聚焦报名—提醒—追踪—跟进的运营编排；不负责具体渠道的消息发送实现；不替代分析报表做跨活动归因。
