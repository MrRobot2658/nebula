# 分析报表 Analytics

> **Skill ID** `analytics-skill` ｜ **分类** 活动 ｜ **文档版本** v0.2

## 一、模块定位
分析报表负责营销活动的效果分析，覆盖全渠道数据，呈现从触达到转化的完整漏斗，并支持跨渠道横向对比与单客户的旅程分析。它把分散在各渠道 Skill、自动化流程、评分模型中的运营数据汇聚成统一的度量视图，让运营用数据回答"活动做得好不好、好在哪、差在哪"，是 Nebula 营销闭环中的"复盘与决策"模块。

该模块解决的核心问题是营销效果的可衡量与可归因：多渠道、多触点的营销链路使得"哪一笔触达带来了转化"难以判断。分析报表通过统一的漏斗模型与多种归因方式，把投入与产出对应起来，支撑预算分配与策略优化。对标 Marketo Analytics、有赞数据中心，强调全渠道打通与可视化自助分析。

## 二、目标用户与典型场景
- **目标用户**：市场分析师、增长运营、营销负责人、运营总监（关注 ROI、渠道效率与策略复盘的决策角色）
- **典型场景**：
  - 活动复盘：大促结束后查看参与人数、触达率、转化率、ROI，定位转化漏斗的瓶颈环节。
  - 渠道对比：横向比较 9 个渠道的触达与转化效果，把预算向高效渠道倾斜。
  - 客户旅程透视：查看单一客户跨多渠道的互动时间线，理解其从认知到成交的真实路径。

## 三、核心能力
| 能力 | 说明 |
| --- | --- |
| 活动概览 | 汇总参与人数、触达率、转化率、ROI 等核心指标，一屏掌握活动整体表现。 |
| 渠道对比 | 对 9 个渠道的效果做横向对比，统一口径评估各渠道触达量、转化率与成本效率。 |
| 转化漏斗 | 呈现"触达 → 参与 → 转化 → 复购"完整漏斗，量化各环节流失，定位瓶颈。 |
| 客户旅程 | 以时间线展示单一客户跨多渠道的互动序列，还原其完整决策路径。 |
| 活动归因 | 支持首次触达、末次触达、线性归因等多种归因模型，灵活分配转化功劳。 |
| 导出 | 支持 PDF / CSV / Excel 多格式导出，便于汇报与二次分析。 |
| 自定义仪表盘 | 拖拽式指标与维度配置，运营可自助搭建专属看板，无需依赖开发。 |

## 四、ChatUI 交互
运营可在对话栏用自然语言提问"上周大促各渠道转化率怎么样"，Agent 直接在对话中以图表卡片回答关键指标，并主动给出 AI 洞察（如"短信渠道转化最高、邮件触达率偏低"）。完整的转化漏斗、渠道对比、自定义仪表盘等复杂报表以右栏面板或独立全屏页呈现，支持拖拽配置指标维度与下钻。查看单个客户时，右栏客户卡片下方的活动时间线即是该客户旅程分析的入口，点击可展开跨渠道互动明细，使分析能力自然融入三栏的"右栏客户上下文"。

## 五、事件与编排
分析报表是事件总线的下游聚合消费者：它不触发营销动作，而是持续汇聚各渠道、流程、评分模块上报的事件，按活动/渠道/客户维度建模并计算指标与归因。

| 归因方式 | 说明 |
| --- | --- |
| 首次触达 | 转化功劳全部归于客户的第一个触点 |
| 末次触达 | 转化功劳全部归于成交前的最后一个触点 |
| 线性归因 | 转化功劳在旅程中各触点间均摊 |

漏斗与旅程的数据来源于事件总线的标准事件（触达、打开、点击、参与、下单、复购等），经 OneID 归一到统一客户身份后串联成跨渠道时间线。

## 六、配置与数据结构
报表以"指标 × 维度 × 时间范围 × 归因模型"配置，仪表盘由组件数组组成：

```json
{
  "dashboardId": "dash_promo_618",
  "scope": { "campaignId": "camp_618", "dateRange": "2026-06-01~2026-06-20" },
  "attribution": "linear",
  "widgets": [
    { "type": "metric", "metric": "reachRate" },
    { "type": "metric", "metric": "conversionRate" },
    { "type": "metric", "metric": "roi" },
    { "type": "funnel", "stages": ["reach", "engage", "convert", "repurchase"] },
    { "type": "channelCompare", "dimension": "channel", "metric": "conversionRate" }
  ]
}
```

## 七、关键指标
- 触达率 / 参与率 / 转化率 / 复购率
- ROI（投入产出比）
- 各渠道效果对比（贡献度排名）
- 漏斗各环节流失率
- 归因后各触点贡献度

## 八、依赖与边界
- **依赖**：统一数据层（CDP / OneID 做客户归一、事件总线提供全渠道事件流）；渠道与内容模块、自动化流程、评分模型（数据来源）。
- **边界**：只做数据聚合、分析与可视化，不触发营销动作、不修改客户数据；不负责实时事件响应（交给触发器）；归因为分析口径，不直接决定预算分配。
