# 自动化流程 Automation

> **Skill ID** `automation-skill` ｜ **分类** 活动 ｜ **文档版本** v0.2

## 一、模块定位
自动化流程是 Nebula 的核心营销编排能力，提供一套可视化拖拽式流程编辑器，让运营人员无需编写代码即可把"触发 → 条件 → 动作"的完整营销链路画成一张可执行的画布。它把分散在各渠道 Skill 中的触达、内容、打分、追踪等原子能力，按照业务剧本编排成端到端的自动化策略，是连接事件总线、CDP/OneID 数据层与各渠道/内容模块的总指挥。

该模块解决的核心问题是营销动作的"人肉化"与"碎片化"：过去运营需要手动判断客户阶段、手动选人群、手动逐渐推送，难以规模化且容易遗漏。自动化流程通过流程化编排，把新客欢迎、流失挽回、生日关怀、大促预热等典型策略沉淀为可复用的资产，实现一次配置、长期自动运转。对标 Marketo Smart Campaign、微盟自动化营销、销售易旅程编排，强调以画布为中心、以事件为驱动的策略编排体验。

## 二、目标用户与典型场景
- **目标用户**：营销运营、增长运营、市场活动经理、用户生命周期运营（负责拉新、促活、留存、转化的策略设计者）
- **典型场景**：
  - 新客欢迎旅程：当客户首次注册/留资时进入流程，依次发送欢迎语、引导关注、推送新人券，并根据是否打开进行分支跟进。
  - 流失挽回：识别 N 天未互动客户，自动进入挽回流程，分批次推送召回内容，未唤醒者升级到人工触达。
  - 大促预热与生日关怀：节假日/生日前按等待节点定时触发多渠道提醒，结合条件判断客户分层差异化推送权益。

## 三、核心能力
| 能力 | 说明 |
| --- | --- |
| 节点类型 | 提供触发、条件、动作、等待、分支五类节点。触发节点定义流程入口（事件/定时/手动），条件节点做属性与行为判断，动作节点调用渠道/内容/标签等能力，等待节点支持固定时长或定时点，分支节点按条件分流形成多路径剧本。 |
| 可视化画布 | 拖拽式流程设计，节点以连线串联，支持缩放、对齐、注释，所见即所得地呈现整条营销链路；画布上可直接看到每个节点的实时运行人数与状态。 |
| 预置模板 | 内置新客欢迎、流失挽回、生日关怀、大促预热等开箱即用模板，运营可一键载入后按需微调，降低从零搭建成本。 |
| 版本管理 | 对流程的每次发布生成历史版本，支持版本对比、追溯与一键回滚，避免误改导致策略中断。 |
| 执行日志 | 记录每个客户在每个节点的执行结果、进入/离开时间与耗时，便于排查为何某客户未收到消息或卡在某节点。 |
| 流程复制 | 支持跨活动复制整条流程或局部子流程作为模板复用，把成熟剧本快速迁移到新活动中。 |

## 四、ChatUI 交互
日常运营在中间对话栏即可发起，例如"帮我搭一个新客欢迎流程"，Agent 会基于模板生成草稿并在对话上下文中给出 AI 建议（推荐节点、推荐人群）。由于流程编排属于复杂可视化配置，画布以右栏面板侧滑展开或独立全屏页呈现，运营在画布上拖拽节点、连线、配置参数；保存后回到对话流，流程的运行概况（进入人数、转化、异常告警）以卡片形式在对话上下文中持续可见。右栏客户卡片可联动显示某个客户当前所处的流程节点，形成"对话发起—画布编排—右栏追踪"的三栏闭环。

## 五、事件与编排
自动化流程是事件总线的主要消费者与动作发起者。流程入口可由事件总线推送的标准事件触发（如某渠道的打开、点击、表单提交、入会等），也可由定时器或人工批量触发。节点编排逻辑如下：

| 节点类型 | 作用 | 示例 |
| --- | --- | --- |
| 触发 | 定义进入流程的事件源/定时/手动 | 监听"表单提交"事件入流 |
| 条件 | 基于 CDP 属性或行为做判断 | 客户行业=金融 且 评分≥80 |
| 动作 | 调用渠道/内容/标签/评分能力 | 发送短信、打标签、加分 |
| 等待 | 延时或等到指定时间点 | 等待 24 小时 / 等到生日当天 |
| 分支 | 按条件分流多路径 | 已打开→路径A，未打开→路径B |

动作节点通过事件总线下发指令给渠道与内容模块（消费渠道事件、触发内容模板），并可写回标签、评分变更等事件，从而与触发器、评分模型形成联动闭环。

## 六、配置与数据结构
关键配置以节点图（DAG）形式存储，每个节点含类型、配置参数与连线关系：

```json
{
  "flowId": "flow_welcome_001",
  "name": "新客欢迎旅程",
  "version": 3,
  "nodes": [
    { "id": "n1", "type": "trigger", "config": { "event": "form.submit" } },
    { "id": "n2", "type": "action", "config": { "channel": "wechat", "templateId": "tpl_welcome" } },
    { "id": "n3", "type": "wait", "config": { "duration": "24h" } },
    { "id": "n4", "type": "condition", "config": { "expr": "event.open == true" } },
    { "id": "n5", "type": "branch", "config": { "true": "n6", "false": "n7" } }
  ],
  "edges": [
    { "from": "n1", "to": "n2" },
    { "from": "n2", "to": "n3" },
    { "from": "n3", "to": "n4" }
  ]
}
```

## 七、关键指标
- 流程进入人数 / 完成人数 / 完成率
- 各节点转化率与流失节点（漏斗瓶颈定位）
- 流程整体转化率与带来的 GMV / ROI
- 节点平均执行耗时与异常率
- 模板复用次数

## 八、依赖与边界
- **依赖**：统一数据层（CDP / OneID / 事件总线）；渠道与内容模块（动作节点实际依赖渠道 Skill 触达、内容模块提供模板）；评分模型与触发器提供条件与入口信号。
- **边界**：不直接负责单渠道的消息发送实现与内容模板创作，只做编排调度；不生产客户数据，人群与属性来自 CDP；不替代触发器做单一实时规则（复杂多步剧本走流程，轻量即时响应走触发器）。
