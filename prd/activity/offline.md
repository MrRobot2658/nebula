# 线下活动 Offline Event

> **Skill ID** `offline-skill` ｜ **分类** 活动 ｜ **文档版本** v0.2

## 一、模块定位
线下活动负责线下会议、展会、门店活动的全流程管理，覆盖活动发布、报名签到、现场互动与会后追访的完整链路。它把线下场景中容易"断链"的数据（谁报名、谁到场、现场逛了哪些展区、留了什么线索）数字化并回流到客户画像，是 B2B 与零售品牌把线下流量转化为可运营线索的关键活动模块。

该模块解决的核心问题是线下营销的数据黑洞：传统线下活动结束后往往只剩一沓名片和模糊的到场印象，线索难以系统化跟进。线下活动通过二维码签到、电子名片、现场互动等手段，把每位参会者的行为转化为事件回流至 CDP，并自动触发会后追访，让线下活动具备与线上同等的可追踪与可运营性。对标 Marketo Event Management、纷享销客活动管理。

## 二、目标用户与典型场景
- **目标用户**：市场活动经理、B2B 市场运营、门店/区域运营、品牌活动策划（负责线下获客与线索转化的角色）
- **典型场景**：
  - 行业展会获客：搭建活动页与票务，参会者扫码签到入场，现场大屏抽奖促互动，会后自动发资料并邀约回访。
  - 门店促销活动：在线报名+扫码核销，现场问卷收集偏好，把到店客户沉淀为可二次触达的会员。
  - 高端客户沙龙：电子名片交换+扫码加好友，追踪嘉宾在各环节的停留，把高意向客户推送给销售追访。

## 三、核心能力
| 能力 | 说明 |
| --- | --- |
| 活动发布 | 支持活动页面搭建与票务设置，对外发布活动并承接报名流量。 |
| 报名管理 | 提供在线报名、报名审核与多渠道确认通知，管理参会名单与准入。 |
| 签到核销 | 支持二维码签到、扫码入场核销，精准记录实际到场情况。 |
| 现场互动 | 内置大屏抽奖、投票、问卷收集等现场互动玩法，活跃气氛并采集意向。 |
| 名片交换 | 支持电子名片与扫码加好友，把线下社交关系沉淀为可触达的数字连接。 |
| 参会者追踪 | 追踪谁来了、去了哪个展区、在哪个环节停留最久，刻画现场行为路径。 |
| 会后跟进 | 自动发送感谢信、资料与回访邀约，把现场热度延续为后续跟进。 |
| 效果分析 | 统计到场率、互动率、线索转化率，量化线下活动的营销产出。 |

## 四、ChatUI 交互
运营可在对话栏说"发布一场下月的城市展会，开放在线报名并配二维码签到"，Agent 生成活动页与票务草稿并在对话中给出报名/确认文案的 AI 建议。活动页搭建、票务设置、签到核销看板、现场互动配置等以右栏面板或独立页呈现；活动当天，实时到场人数、签到进度、互动参与以状态卡片在对话上下文滚动可见，运营可随时掌握现场态势。右栏客户卡片在查看某参会者时，联动展示其报名、签到、停留展区与名片交换记录，让线下行为成为客户上下文的一部分，呼应三栏理念。

## 五、事件与编排
线下活动向事件总线产出从报名到追访的全流程事件，并消费事件驱动确认与跟进。

| 事件 | 触发的编排 |
| --- | --- |
| `offline.register`（报名） | 多渠道确认通知、加入会前提醒 |
| `offline.checkin`（扫码签到） | 标记到场、更新出席状态 |
| `offline.interact`（抽奖/投票/问卷） | 累加评分、采集意向标签 |
| `offline.card`（名片交换/加好友） | 建立数字连接、回流社交关系 |
| `offline.zone`（展区停留） | 记录兴趣点、刻画行为路径 |
| `offline.end`（结束） | 自动发感谢信、资料、回访邀约 |

事件回流总线后，可被评分模型加分、被触发器命中、被自动化流程作为入口，使到场与现场行为驱动后续精准追访。

## 七、关键指标
- 报名数 / 到场率（实到/报名）
- 现场互动率（抽奖/投票/问卷参与度）
- 名片交换/加好友数
- 各展区停留分布与热点环节
- 线索转化率与会后追访响应率

## 八、依赖与边界
- **依赖**：统一数据层（CDP / OneID / 事件总线）；渠道与内容模块（确认通知、会后跟进依赖短信/微信/邮件渠道与内容模板）；评分模型、触发器、自动化流程（消费线下事件做后续编排）。
- **边界**：不负责线下场地、票务支付等线下履约实现，聚焦报名—签到—互动—追访的数字化运营编排；不负责具体渠道的消息发送实现；不替代分析报表做跨活动综合归因。
