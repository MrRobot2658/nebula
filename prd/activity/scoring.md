# 评分模型 Scoring

> **Skill ID** `scoring-skill` ｜ **分类** 活动 ｜ **文档版本** v0.2

## 一、模块定位
评分模型负责对客户进行量化打分，帮助运营在海量客户中快速识别高意向、高价值客户，把有限的营销与销售资源投向最值得跟进的人群。它基于行为、属性、互动三个维度动态计算客户分值，当评分达到预设阈值时自动触发后续动作或打标签，实现"识别即行动"的闭环。评分是连接客户数据与营销决策的关键中间层。

该模块解决的核心问题是客户分层与优先级判断：仅靠人工经验难以持续、客观地评估每个客户的成熟度。通过把行为信号（浏览/点击/购买）、静态属性（行业/职级/规模）与互动质量（咨询/投诉/好评）折算成可比的分数，运营得以用统一标尺衡量线索质量并自动分级。对标 Marketo Lead Scoring、销售易客户评分。

## 二、目标用户与典型场景
- **目标用户**：增长运营、市场线索运营（MOps）、销售运营、SDR/销售团队（关注线索质量与跟进优先级的角色）
- **典型场景**：
  - 线索分级派发：新客初始 60 分，随互动累加，达 80 分自动触发"高意向客户"规则并派发给销售优先跟进。
  - 沉默客户降权：长期未互动的客户分值随衰减机制逐渐下降，自动移出重点跟进池，节省触达成本。
  - 风险预警：当"投诉"事件导致互动分骤降时，自动打"需安抚"标签并触发客户成功介入。

## 三、核心能力
| 能力 | 说明 |
| --- | --- |
| 三维度评分 | 行为分（浏览/点击/购买等行为信号）+ 属性分（行业/职级/企业规模等静态画像）+ 互动分（咨询/投诉/好评等互动质量），三维加权合成总分，立体刻画客户成熟度。 |
| 规则配置 | 按事件配置增减分规则，支持为不同事件设置权重，如"打开邮件 +5""完成购买 +20""投诉 -15"。 |
| 阈值触发 | 当总分或某维度分达到阈值时，自动触发动作（通知、进入流程）或打标签，实现评分到行动的自动转化。 |
| 衰减机制 | 对长期未互动客户的评分按时间逐渐衰减，避免历史高分掩盖当前低活跃，保持分值的时效性。 |
| 可视化 | 提供评分趋势图（单客户分值随时间变化）与分布图（客户群分值分布），辅助运营理解人群结构。 |
| 历史追溯 | 记录每次评分变化的来源事件、增减值与时间，形成完整评分变更日志，可追溯任一分值的成因。 |

## 四、ChatUI 交互
运营可在对话栏直接询问"这个客户为什么是 85 分"，Agent 调取评分变更日志在对话上下文中给出逐条加减分解释。评分模型本身的配置（维度权重、各事件分值、阈值、衰减系数）属于结构化复杂配置，以右栏面板或独立页的"评分模型编辑器"呈现，运营在此调整权重并实时预览分布变化。趋势图与分布图等可视化以右栏面板展开。右栏客户卡片常驻显示当前客户的总分、三维度分解与近期变化趋势，使评分成为查看任一客户时的默认上下文，呼应三栏理念。

## 五、事件与编排
评分模型订阅事件总线上的行为与互动事件，按规则实时增减分，并在跨越阈值时对外触发动作。

| 环节 | 内容 |
| --- | --- |
| 输入事件 | 渠道行为事件（浏览、点击、购买）、互动事件（咨询、投诉、好评）、属性变更事件 |
| 计算 | 按事件→分值映射与权重累加/扣减，叠加衰减 |
| 触发动作 | 达阈值时：打标签、进入自动化流程、通知销售、触发触发器规则 |
| 回写 | 评分变更作为事件回写总线，供触发器/流程/分析消费 |

示例：新客初始 60 分，一次互动 +5、深度互动 +20，累计达到阈值 80 时触发"高意向客户"规则；评分变更同时回写事件总线，触发器据此可进一步联动派发动作。

## 六、配置与数据结构
关键配置包括维度权重、事件分值表、阈值与衰减参数：

```json
{
  "modelId": "score_default",
  "dimensions": {
    "behavior": { "weight": 0.4 },
    "attribute": { "weight": 0.3 },
    "interaction": { "weight": 0.3 }
  },
  "rules": [
    { "event": "page.view", "dimension": "behavior", "delta": 2 },
    { "event": "order.create", "dimension": "behavior", "delta": 20 },
    { "event": "consult", "dimension": "interaction", "delta": 5 },
    { "event": "complaint", "dimension": "interaction", "delta": -15 }
  ],
  "initialScore": 60,
  "thresholds": [
    { "score": 80, "action": "tag:高意向客户" }
  ],
  "decay": { "idleDays": 30, "ratePerWeek": 5 }
}
```

## 七、关键指标
- 高分客户数 / 高分客户转化率
- 各维度平均分与分值分布
- 达阈值触发次数及触发后转化率
- 衰减影响人数（被降权客户量）
- 评分准确性（高分客户实际成单率回测）

## 八、依赖与边界
- **依赖**：统一数据层（CDP / OneID 提供客户属性与统一身份、事件总线提供行为与互动事件）；渠道与内容模块（事件来源）；触发器与自动化流程（达阈值后的动作执行方）。
- **边界**：只负责打分与阈值判定，不直接发送消息或执行触达（交给渠道/流程）；不定义客户身份与去重（由 OneID 负责）；不替代业务做最终跟进决策，仅提供量化参考。
